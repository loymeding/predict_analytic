# Лабораторная работа №3 - Классификация временных рядов и прогнозирование метеоданных

## Описание

Двухэтапная система анализа метеоданных шести российских городов:
1. **Классификация** - по 30-дневному временному ряду определить климатический тип (MiniROCKET)
2. **Прогнозирование** - предсказать температуру на следующие 30 дней (Holt-Winters)

## Данные

Папка `данные/` - почасовые метеоданные за 2019-2025 гг. для городов:
- Черноморский климат: Сочи, Геленджик
- Умеренно-континентальный: Москва, Санкт-Петербург
- Дальневосточный: Благовещенск, Находка

Каждый файл содержит: `temperature_2m`, `relative_humidity_2m`, `precipitation`, `rain`, `snowfall`, `weathercode`, `wind_speed_10m`, `surface_pressure`

## Запуск

1. Установить зависимости:
```
pip install -r requirements.txt
```

2. Открыть ноутбук:
```
jupyter notebook lab3.ipynb
```

3. Запустить все ячейки (Kernel -> Restart & Run All)

> Классификация (MiniROCKET) занимает ~20 секунд. Прогнозирование ~4 секунды на город.

## Структура ноутбука

| Раздел | Содержание |
|--------|-----------|
| 2.1 EDA | Визуализация телеметрии, сезонные паттерны, корреляции, тест ADF |
| 2.2 Признаки | Временные, циклические, лаговые, PCA разделимости |
| 2.3 Классификация | MiniROCKET (sktime), обоснование выбора |
| 2.4 Оценка классификации | Accuracy/F1/ROC-AUC, матрица ошибок, важность каналов |
| 2.5 Прогнозирование | Holt-Winters per-city, train 2019-2023, test янв.2025 |
| 2.6 Оценка прогноза | MAE/RMSE/MAPE, анализ остатков, визуализация |
| 2.7 Пайплайн | Классификация + прогноз в одной функции |
| 2.8 Выводы | Сводная таблица результатов |

## Интерпретация результатов

- **Классификатор Accuracy = 1.0** - климатические зоны очень контрастны по температуре, 30 дней данных достаточно для однозначного разделения
- **MAE прогноза 3-6°C** - типичная точность для 30-дневного горизонта; лучше у стабильных морских климатов (Сочи), хуже у резко-континентальных (Благовещенск)
- **ConvergenceWarning** при Holt-Winters - известная особенность оптимизатора statsmodels при seasonal_periods=365, результаты корректны
