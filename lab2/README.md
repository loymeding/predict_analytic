# Лабораторная работа №2 - Виртуальный датчик для печи обжига

## Описание

Задача - предсказать концентрацию выходного продукта печи обжига по данным телеметрии, не дожидаясь лабораторного анализа, который приходит с задержкой 10-15 минут и нерегулярно.

## Данные

| Файл | Описание |
|------|----------|
| `data_train.csv` | Минутная телеметрия (16 датчиков) за 8 месяцев, обучение |
| `target_train.csv` | Лабораторные замеры концентрации, обучение |
| `data_test_small.csv` | Телеметрия для теста |
| `target_test_small.csv` | Замеры для теста |

## Запуск

1. Установить зависимости:
```
pip install -r requirements.txt
```

2. Открыть ноутбук:
```
jupyter notebook lab2.ipynb
```

3. Запустить все ячейки последовательно (Kernel -> Restart & Run All)

## Структура ноутбука

- **2.1 EDA** - анализ телеметрии и лабораторных замеров, синхронизация источников, корреляции
- **2.2 Feature Engineering** - лаговые признаки, скользящие статистики, производные
- **2.3 Модели** - Ridge Regression, Random Forest, CatBoost
- **2.3 Оценка** - MAE, RMSE, MAPE, WAPE, анализ остатков, SHAP, AIC/BIC
- **2.4 Выводы** - интерпретация результатов

## Интерпретация результатов

Модель предсказывает концентрацию целевого продукта в текущий момент. Чем ниже MAE/RMSE - тем точнее. По SHAP видно какие датчики и с каким лагом влияют на качество продукта - это помогает инженерам понять физику процесса.
